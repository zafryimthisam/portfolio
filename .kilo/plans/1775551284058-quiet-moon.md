# Plan for Adding Image Background Remover Tool

## Theme Analysis from CollageMaker

From analyzing the CollageMaker component and the overall site theme:

- **Color Scheme**: Dark theme with `bg-neutral-950` (very dark gray), `text-white`, containers `bg-black` with `border-neutral-700`
- **Typography**: `font-mono`, headings with gradient text (e.g., `from-amber-200 to-yellow-500 bg-clip-text text-transparent`), large bold fonts
- **Layout**: Sidebar + main content using `grid xl:grid-cols-[380px_1fr]`, responsive with md: prefixes
- **Components**: Rounded corners (`rounded-xl`, `rounded-2xl`), shadows (`shadow-xl shadow-black/30`), hover effects with amber accents
- **Buttons**: Primary: `bg-amber-500 text-black hover:bg-amber-400`, Secondary: `border border-neutral-700 bg-neutral-900`
- **Inputs**: `rounded-xl border border-neutral-800 bg-neutral-900 text-white`
- **Upload Areas**: `rounded-2xl border border-dashed border-neutral-700 bg-neutral-900/70`
- **Spacing**: `p-4 md:p-5`, `space-y-4`, `gap-4`
- **Icons**: Tabler icons, minimal use of emojis in some places
- **Responsive**: Mobile-first with md: breakpoints

The overall feel is modern, minimal, professional dark UI with amber/yellow accents for highlights.

## Implementation Plan

### 1. Update Tools Listing Page
- Add a new card in `app/tools/page.tsx` for "Background Remover" following the same style as CollageMaker card
- Use an appropriate icon (e.g., 🖼️ or similar), amber background for the icon badge

### 2. Create Tool Component
- Create `components/web/ImageBackgroundRemover.tsx` matching CollageMaker's structure and styling
- Use similar layout: sidebar for controls, main area for preview
- Include:
  - Upload area for image input
  - Preview of original and processed image
  - Remove background button
  - Reset button
  - Download button for processed image
  - Loading states and error handling
  - Status/progress indicators

### 3. Create API Route
- Create `app/api/background-remove/route.ts`
- Handle POST requests with FormData containing the uploaded image
- Use `@remove-background-ai/rembg.js` with API key from `process.env.REMBG_API_KEY`
- Support output formats: webp and png (default webp for smaller size)
- Return the processed image as response
- Error handling for API failures

### 4. Update Tool Router
- Modify `app/tools/[tool]/page.tsx` to conditionally render the appropriate component based on the tool slug
- Add support for "background-remover" slug

### 5. Environment Setup
- Ensure `REMBG_API_KEY` is documented (user will need to add this to their env vars)

### 6. Testing and Validation
- Test with various image formats and sizes
- Ensure Vercel deployment compatibility (no Node.js specific issues)
- Run linting and type checking

## Technical Details

### API Route Requirements
- Accept multipart/form-data with image file
- Use rembg to process the image
- Return processed image with appropriate content-type
- Handle errors gracefully

### Frontend Requirements
- File upload with drag & drop (similar to CollageMaker)
- Preview both original and result
- Loading spinner during processing
- Download functionality for result
- Reset to start over

### Matching CollageMaker Style
- Use identical class names and structure where possible
- Maintain consistent spacing, colors, and interactions
- Use same progress/status patterns
- Follow the same button and input styling

This plan ensures the new tool seamlessly integrates with the existing design language and functionality.</content>
